# Requisitos Não Funcionais (RNF)

- **RNF01:** A aplicação deve ser desenvolvida em ambiente Flutter para garantir performance nativa.

- **RNF02:** A autenticação e persistência de sessão devem ser geridas via Firebase/Google Auth.

- **RNF03:** A interface deve seguir os princípios de UX para dispositivos móveis, garantindo que a edição de perfil seja intuitiva.

- **RNF04:** A aplicação deve ser capaz de lidar com requisições externas de forma assíncrona, especialmente na comunicação com as APIs de CEP e Mercado Pago.

- **RNF05:** As transações financeiras realizadas via API do Mercado Pago devem ocorrer sob protocolos de comunicação segura.
